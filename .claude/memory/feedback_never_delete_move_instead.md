---
name: Never delete data — move it instead
description: When reorganizing files, configs, or memories, MOVE content to the new location. Never delete/scrub the original without preserving it somewhere.
type: feedback
---

When told to move or reorganize content, MOVE it — don't delete it. The original data must be preserved somewhere accessible. Deletion is a last resort that requires explicit user confirmation.

**Why:** Agents default to "clean up" by deleting, which destroys data the user may need. Moving preserves everything while achieving the reorganization goal.

**How to apply:** When migrating files, memories, configs — copy to the new location first, verify it's there, then only remove the original if explicitly asked. When splitting content (e.g., project-specific vs general), move each piece to its correct new home. Never discard.
